# Desktop releases (S3 + authenticated API)

Production desktop updates use a **private S3 bucket** for artifacts and an **authenticated HTTP feed** on your platform API (`BASE_API`). The GitHub repository stays private; end users never receive publish credentials.

## Architecture

```
GitHub Actions (Release workflow)
  AWS credentials (GitHub secrets, write-only)
       │
       ▼
  s3://<bucket>/desktop/releases/stable/
       ├── latest-mac.yml
       ├── latest.yml
       ├── OpenFDE-<version>-mac.zip
       └── OpenFDE Setup <version>.exe

Installed OpenFDE app (signed in)
  GET {BASE_API}/desktop/releases/stable/latest-mac.yml
      Authorization: Bearer <server-jwt>
       │
       ▼
  Your API validates JWT → streams object from S3 (or presigned redirect)
```

## S3 layout


| Key                                                | Purpose                           |
| -------------------------------------------------- | --------------------------------- |
| `desktop/releases/stable/latest.yml`               | Windows feed (`electron-updater`) |
| `desktop/releases/stable/latest-mac.yml`           | macOS feed                        |
| `desktop/releases/stable/latest-linux.yml`         | Linux feed (when built)           |
| `desktop/releases/stable/*.{exe,zip,dmg,blockmap}` | Installers referenced by feed     |


Feeds and binaries must **not** be public-read. Only your API (with IAM read access) should reach the bucket.

Optional staging channel: `desktop/releases/sit/` — override `app.desktop.releasesUrl` in `env/.sit.env`.

## GitHub Actions secrets

Configure in the repository **Settings → Secrets**:


| Secret                  | Purpose                                           |
| ----------------------- | ------------------------------------------------- |
| `AWS_ACCESS_KEY_ID`     | CI upload (S3 `PutObject` on release prefix only) |
| `AWS_SECRET_ACCESS_KEY` | CI upload                                         |
| `AWS_REGION`            | Bucket region                                     |
| `S3_RELEASE_BUCKET`     | Bucket name                                       |


The Release workflow runs `npm run release:upload-s3` after building installers.

Local upload (after a build):

```bash
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
export AWS_REGION=us-east-1
export S3_RELEASE_BUCKET=your-bucket
npm run release:mac    # or release:win
npm run release:upload-s3
```

## Client configuration

In `env/.prod.env` (maps to `app.base.apiUrl`):

```properties
BASE_API = 'https://api.example.com'
```

Default update feed: `{BASE_API}/desktop/releases/stable/`

Optional override in `config.properties`:

```properties
app.desktop.releasesUrl=desktop/releases/stable
```

The desktop app calls `autoUpdater.setFeedURL({ provider: 'generic', url: feedUrl })` at runtime and sends `Authorization: Bearer <server-access-token>` (same Google sign-in flow as support upload).

Users must be **signed in** to check or download updates.

## Server API contract (implement on your backend)

All routes require a valid platform JWT (`Authorization: Bearer …`), same validation as `/support/upload`.

### Feed metadata

```
GET /desktop/releases/stable/latest-mac.yml
GET /desktop/releases/stable/latest.yml
GET /desktop/releases/stable/latest-linux.yml
```

Return the YAML bytes from S3 with `Content-Type: application/x-yaml`.

Example feed (generated by electron-builder — do not hand-edit):

```yaml
version: 0.0.2
files:
  - url: OpenFDE-0.0.2-mac.zip
    sha512: <base64>
    size: 198765432
path: OpenFDE-0.0.2-mac.zip
sha256: <base64>
releaseDate: '2026-06-28T10:00:00.000Z'
```

`url` paths are relative to the feed base URL. The app requests them with the same Bearer token.

### Installer binaries

```
GET /desktop/releases/stable/<filename>
```

Stream from S3 or respond with `302` to a short-lived presigned URL.

### Responses


| Status | Meaning                          |
| ------ | -------------------------------- |
| `401`  | Missing/invalid JWT              |
| `404`  | Object not in S3 / wrong channel |
| `200`  | Feed or installer body           |


Anonymous access must return `401` for every path under `/desktop/releases/`.

## IAM sketch

**CI user (publish only):**

```json
{
  "Effect": "Allow",
  "Action": ["s3:PutObject", "s3:PutObjectAcl"],
  "Resource": "arn:aws:s3:::your-bucket/desktop/releases/*"
}
```

**API server (read only):**

```json
{
  "Effect": "Allow",
  "Action": ["s3:GetObject"],
  "Resource": "arn:aws:s3:::your-bucket/desktop/releases/*"
}
```

Never embed CI credentials in the desktop app.
